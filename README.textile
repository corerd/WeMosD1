h2. Using GitHub Pages

On any regular repository, the *gh-pages* branch has a special meaning for GitHub: any content pushed here, will automatically published to the project GitHub Pages.

The instructions can be found at "Creating Project Pages manually":https://help.github.com/articles/creating-project-pages-manually.

Since "git 1.7.10":https://lkml.org/lkml/2012/3/28/418 (April 2012) git allows you to clone only one branch by means of the new `--single-branch` option.


h2. Folder structure

/path/to/WeMosD1-workspace                # parent of "gh-pages" and "master" subfolders
/path/to/WeMosD1-workspace/gh-pages
/path/to/WeMosD1-workspace/gh-pages/.git  # checkout of "gh-pages" branch (removed "master" branch)
/path/to/WeMosD1-workspace/WeMosD1
/path/to/WeMosD1-workspace/WeMosD1/.git   # checkout of "master" branch


h2. Cloning

cd /path/to/WeMosD1-workspace

With HTTPS URL
git clone -b gh-pages --single-branch https://github.com/corerd/WeMosD1.git gh-pages  # clone "gh-pages" branch
git clone -b master --single-branch https://github.com/corerd/WeMosD1.git             # clone "master" branch

With SSH URL
git clone -b gh-pages --single-branch git@github.com:corerd/WeMosD1.git gh-pages  # clone "gh-pages" branch
git clone -b master --single-branch git@github.com:corerd/WeMosD1.git             # clone "master" branch
